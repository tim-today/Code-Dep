# 修改日志

## 2026-06-04
- 优化: 引入全局加载状态阻断机制。在系统首次成功加载 `/api/bootstrap` 基础数据之前，阻断渲染表单或触发路由交互，展示统一的 Loading 界面，防止用户在数据未就绪时进行操作。
- 优化: 实现下拉框防御性回显与保护机制。重构密钥、节点及通知等下拉菜单渲染函数，当传入的当前值暂不在可选列表时，强制生成一个隐藏的占位 option，防止浏览器自动将其重置为空值并在保存时覆盖清空原有敏感配置。
- 优化: 引入保存防重及置灰指示。在项目配置保存与全局 modal 保存异步提交期间，将按钮设为 `disabled` 并展示 `正在保存...`，防止网络延迟时重复点击及竞态冲突。
- 新增: 在 `i18n.js` 中补充 `saving` 和 `loading` 的多语言翻译词条。
- 优化: 美化物理节点控制台的终端提示信息。重构了 WebSocket 的各生命周期回调，通过 ANSI Escape Code 在 xterm.js 终端内部输出彩色且带图标（如闪电、绿勾、红叉）的中英文连接提示，直观呈现连接中、连接成功、断开连接及报错等交互信息。
- 优化: 更新 `index.html` 中的前端静态资源引入版本号（Cache Buster），解决浏览器强缓存导致新代码无法立即生效的问题。
- 优化: 美化 Go 后端主程序启动时的终端提示。将原本单调的单行日志改为包含炫酷 ASCII 艺术字 Banner、高亮绿勾状态、Web UI 访问地址、数据目录绝对路径及默认管理员提示等彩色分类回显，显著增强了系统启动的专业视觉感。

## 2026-06-03
- 优化: 节点管理中编辑 / 新增节点的分组输入框重构为 Tag 多选下拉 Picker（参考编译器选择与目标分组选择体验）。支持勾选已存节点分组以 Tag 形式展示与快速删除；同时支持输入过滤，当输入不存在的分组时，在下拉框最顶部提供「💡 添加分组 "xxx"」的智能交互，极大改善了分组录入的体验。
- 优化: 修复物理节点因配有多分组而在列表下被重复渲染的 Bug。重构了前端节点表格的展现逻辑，去除按分组 map 节点的分节模式，改为直接遍历各物理节点（每节点只渲染一行），并在“分组”一列中支持以逗号拆分出多个独立 Badge（标签 / tag）并排展示，消除了重复行并优化了体验。
- 新增: 节点管理的分组属性支持以中英文逗号多选。重构了后端多分组规范化 `normalizeMultiNodeGroup` 和交集权限匹配，重构了前端 `splitGroup` 辅助函数以支持多分组展开、页面过滤 and 节点数正确统计，并向下兼容旧有单分组老数据。
- 新增: 发布界面节点部署进度与实时状态可视化展示。后端在 `deployArtifacts` 开始时打印 `目标部署节点列表`，且在每个节点开始、成功、失败后分别输出特定的标准化进度日志；前端提取 SSE 管道日志行，基于正则匹配实时解析部署进度及各个节点状态（支持 waiting、pending、ok、failed），并配合 emoji 小图标及呼吸动画，在发布日志监控弹窗顶部进度条旁以 Fraction (如 `1/3`) 和列表形式实时刷新。
- 修复: 命令行编辑器中输入文字与光标不一致的 Bug。通过重构 `web/static/style.css` 的 `.shell-editor` 规则，将 `pre`、`textarea` 和 `code` 的字体、字号、整数行高、tab-size 完全统一，并禁用了 `textarea` 的 `resize` 属性和聚焦时的投影干扰，保证两层内容完美重合对齐。
- 恢复: 节点管理中的节点列表受用户配置的节点分组授权进行过滤的逻辑。若用户未分配任何节点授权则默认不受限制，若有配置则只展示被授权分组的节点；而编译器管理（选择编译器）则依然保持不受该限制。
- 修复: 支持通过环境变量 `DATA_DIR` 自定义数据持久化目录，并支持通过 `DATA_KEY` 环境变量配置固定 AES 密钥（自动通过 SHA-256 转换为 32 字节），彻底解决升级版本时因随机 `.key` 文件丢失或数据路径变更导致的账号和节点信息失效、丢失问题。
- 修复: 解决发布新版后日志控制台内容为空的 Bug。修复了因 `#publishModal` 关闭事件异步排队触发，导致刚刚重新开启的实时日志流被错误关闭并清空当前记录对象的竞态问题。
- 优化: 编译器选择与显示不再受权限限制。取消了获取编译器（Workers）与节点（Nodes）列表时的用户分组过滤，所有人均可查看及可选所有编译器，并在发布执行时不再限制编译器所属节点权限。
- 修复: 移除已被删除的 `#copyRawLog` 按钮的事件监听注册，彻底解决导致 `TypeError: Cannot read properties of null` 的运行时错误。
- 优化: 重构 Raw 日志逻辑。彻底删除了独立的 Raw 日志弹窗，将功能按钮更名为“完整日志”并放置于日志底部的操作按钮行最右侧。点击后直接在当前日志文本区中加载并展现完整的构建日志，不用弹出多余的页面。
- 修复: 将发布历史的操作菜单重构为 position: fixed 并使用 JS 动态根据按钮位置计算 fixed 坐标，彻底消除局部列表 overflow 对其的裁剪和撑大滚动条。
- 优化: 将发布表单弹窗（表单模式）高度固定为 380px 并将 overflow 设置为 visible，确保编译器、分支/Tag 下拉菜单能优雅地悬浮穿透到弹窗外部，不再影响和撑大弹窗自身的物理高度。
- 优化: 在表单模式下将表单网格容器重置为单列 1fr，使发布表单以 4 行全宽排版呈现，确保当用户在字段内选中多项内容时（例如分支、多编译器等标签）有足够开阔的横向显示空间，彻底解决撑高变形问题。
- 优化: 将项目发布表单中的“项目”、“编译器”、“环境”、“分支/Tag”由单行改造成两两并排的多行网格布局，改善表单各输入项在窄弹窗下的局促排版。
- 修复: 在表单模式下将发布弹窗表单容器的 Grid 布局重置为自适应高度的 Flex 纵向布局，解决隐藏滚动日志后按钮和弹窗被拉伸拉长的问题。
- 优化: 调整项目发布表单交互，去除表单的控制台日志滚动区域并减小弹窗尺寸（表单模式），点击开始发布后自动关闭表单，并在大尺寸下（日志模式）拉起对应的发布任务日志监控弹窗。
- 修复: 限制旧版卡片样式选择器仅在 .perm-editor 容器下生效，消除其对新版表格 tr.perm-row (display: grid 导致的内容挤在一列) 的布局干扰。
- 优化: 将用户编辑弹窗中“项目授权”和“节点分组授权”重构为 table + 细线布局，使用统一表头且行末只保留 checkbox，使界面整体更紧凑，避免样式重叠和被挤压。
- 优化: 用户编辑弹窗授权区域改为项目信息与权限按钮分区布局，移除横向滚动和拥挤错位。

- 更新: 项目授权为空、节点分组授权为空时默认全部开放。
- 更新: 节点/worker 列表与编辑、项目列表与编辑/创建、发布环境与发布 worker 选择均接入统一权限过滤。

- 更新: 节点新增分组属性，新增/编辑节点时可设置，空值默认归为“默认分组”。
